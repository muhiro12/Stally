# User Workflows

## Start From An Empty Library

The empty state should help the user begin without pressure.

Preserved entry points are:

- Add Your First Item.
- Try Sample Items.
- Restore From Backup.

The onboarding language should keep the original product focus:

- Start with a few pieces the user actually reaches for.
- Clothing, shoes, bags, notebooks, or one small other category are enough to
  begin.
- Add an item, mark it once when it was chosen today, and let the accumulation
  build softly over time.

Sample items should be safe exploration material for an empty library. They
should not be treated as the user's real collection.

## Add And Describe An Item

The user creates an item with a required name and category.

Optional context includes a note and a photo. Notes and photos are supporting
context, not required setup.

The editor workflow should preserve this intent:

- Add a new item.
- Edit item basics later.
- Preserve existing marks while changing basics.
- Delete an item only with clear warning that its marks are removed too.

## Mark Today

The simplest recurring workflow is:

1. Open an item from Library.
2. Mark it for today when the user chose it.
3. Undo today's mark if it was accidental.

The product should continue to frame this as a small daily signal. One mark is
enough for a day.

## Adjust History

The user can correct a past day without changing the rest of the item.

Preserved language:

- Adjust History.
- Adjust Another Day.
- Review Another Day.
- Add Mark.
- Remove Mark.
- Marked.
- Not marked.

This workflow should feel careful because it changes history.

## Browse And Refine The Library

The Library should support scanning the active collection and finding items
through search, category, quick filters, and sort options.

Preserved refinements include:

- Search items.
- All Categories.
- Open Today.
- Marked Today.
- Never Marked.
- With History.
- Without History.
- Recently Marked.
- Most Marked.
- Name.

The user should be able to reach item links, settings, add item, Review,
Archive, Insights, and Backup Center from product entry points, but the old
navigation shape is not a preservation requirement.

## Use Archive

Archive stores past favorites without losing their history.

The user workflows are:

- Move an active item into Archive.
- Browse archived items separately.
- Search or filter archived items.
- Move an archived item back to Library.
- Copy an item link from Archive.

Preserved archive framing:

- Past favorites can stay nearby without crowding the main list.
- Archived items will wait until the user moves them back into Library.
- Archived items stay out of Library until restored.

## Review Items

Review gathers attention lanes:

- Needs First Mark.
- Dormant.
- Recovery Candidates.

The user workflows are:

- Open Review when the list starts to drift.
- Inspect items waiting for a first mark.
- Inspect items whose last mark feels far enough away to revisit.
- Inspect archived items whose history suggests they may deserve another turn.
- Archive untouched or dormant items.
- Move recovery candidates back to Library.
- Select several items in one lane and apply the same next step.

Review should stay gentle. It identifies items deserving attention; it should
not shame the user for inactivity.

## Read Insights

Insights helps the user read the collection as a pattern.

The user workflows are:

- Choose a reading range: 30 Days, 90 Days, 365 Days, or All Time.
- Decide whether archived items should be included.
- Review activity, consistency, categories, rankings, rhythm, and next moves.
- Open a suggested item.
- Copy or share the generated report.
- Copy a link to Insights.

All metrics should follow the selected window.

## Back Up And Restore

Backup Center groups higher-risk collection operations.

The user workflows are:

- Export Backup.
- Choose Backup File.
- Preview the imported backup.
- Merge Into Library.
- Replace Library.
- Delete Everything when intentionally starting over or testing restore.

Preserved safety rules:

- Export before higher-risk changes.
- Keep one recent export before replace-style restore.
- Preview how many items would merge, replace, or be rejected.
- Make merge and replace consequences clear.
- Make Delete Everything explicit and destructive.

## Share Links

The product supports link sharing for:

- App destinations such as Library, Archive, Backup Center, Insights, Review,
  Create Item, and Settings.
- Individual items from item cards and detail.

Unsupported links should show a clear unsupported-link message.
